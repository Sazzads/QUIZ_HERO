1. Difference between Map,filter,ForEach and Find
Map ForEach filter Find
produces a new array with the results of calling the specified function on each member in the called array. For each array element, the supplied function is executed once. The filter() function is used to filter all of the items and returns the element that matches while removing the elements that do not match. The find() function is used to locate all descendent elements of the currently chosen element. It locates the element in the DOM tree by traversing from root to leaf.
map is a method that, like callback, accepts a callback function and returns a new array with the same number of entries as the original array. forEach is a method that accepts a callback function and calls it once for each array element. A filter method returns a new array containing all of the original array's items that fulfill a specific criterion. The find function retrieves the first member of an array that meets a particular criterion.
The map function transforms each element of an array and returns a new array with the modified values. forEach is a function that performs an operation on each array element. The filter function generates a new array containing all entries that fulfill a criterion. The find function is used to return the first element that meets a criterion.
2. difference between local storage, session storage and cookies
local storage session storage cookies
Local storage is a method of storing client-side data that survives even after the web window is closed. The method is a key-value pair storage technique that uses strings to store information. Session storage is a method of storing data on the client side that lasts only as much as the browsers window or tab is open. It, like local storage, is a key-value pair storage method that saves data as strings. Each HTTP request includes the transmission of a cookie, which is a short text file saved just on client and sent to the server. User preferences, session data, as well as other data can all be stored in cookies.
Local storage has a capacity of 5MB/10MB. Session storage has a capacity of 5MB. Cookies have a storage capacity of 4KB.

3. what is event loop and how is it work?

Event-Loop: The concurrency paradigm of JavaScript, which enables the language to handle numerous activities concurrently without obstructing the execution of other processes, is fundamentally based on the event loop.

An ongoing process known as the event loop in JavaScript listens for events like user input, network requests, and timer events and processes them in the order they are received. An event is added to the "task queue" when it takes place. The task queue is continually checked by the event loop for any pending tasks, which are then carried out one by one in the order they were received.

Work Process:

  • 1. The primary thread of execution begins running the code and synchronously carrying out each function one at a time. Functions are added to the call stack as they are invoked.
  • 2. The related callback function is registered with the event loop and added to a task queue whenever an asynchronous event takes place, such as a user pushing a button, a network request concluding, or a timer ending. The event loop waits until the call stack is empty before executing the callback function.
  • 3. The task queue is continually checked by the event loop for unfinished work. The event loop pulls the first job from the task queue and pushes its callback function onto the call stack, where it is performed, when the call stack is empty.
  • 4. The event loop keeps processing asynchronous tasks as they are added to the task queue one at a time, always waiting for the call stack to be empty before moving on to the next job.
  • 5. The event loop keeps checking the task queue for new tasks after a function has completed running by removing it from the call stack.

The event loop allows JavaScript to handle asynchronous activities effectively by continually monitoring a task queue for outstanding jobs and executing their associated callback functions in a non-blocking, single-threaded manner.
